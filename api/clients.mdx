## Overview

The Clients API allows you to manage hotel guest information including names, email addresses, and phone numbers. All client operations are performed on the `cliente` database table.

**Base endpoint**: `/clientes`

---

## Get All Clients

<ParamField path="GET" query="/clientes" required>
  Retrieve a list of all registered clients in the system.
</ParamField>

### Request

```bash
GET http://localhost:4000/clientes
```

### Response

Returns an array of client objects.

<ResponseField name="id" type="integer">
  Unique identifier for the client
</ResponseField>

<ResponseField name="nombre" type="string">
  Client's full name
</ResponseField>

<ResponseField name="correo" type="string">
  Client's email address
</ResponseField>

<ResponseField name="telefono" type="string">
  Client's phone number
</ResponseField>

### Example

<CodeGroup>
```bash cURL
curl http://localhost:4000/clientes
```

```javascript JavaScript

```

## Error Handling

<Warning>
All endpoints will throw database errors if operations fail. The API currently does not implement structured error handling.
</Warning>

Common error scenarios:
- Database connection failures
- Invalid SQL queries
- Constraint violations (e.g., deleting a client referenced in reservations)

### Example Error

If a database error occurs, you'll receive a 500 status code with the error details:

```json
{
  "code": "ER_NO_SUCH_TABLE",
  "errno": 1146,
  "sqlMessage": "Table 'database.cliente' doesn't exist",
  "sqlState": "42S02"
}
```

## Implementation Details

<Expandable title="View Source Code Reference">

The Clients API is implemented in the following files:

**Router**: `reservaionFronEnd/router/clientes.Router.js`
```javascript
router.get('/clientes', service.obtenerListClientesService);
router.post('/clientes', service.addDataCliente);
router.put('/clientes/:id', service.updateDataCliente);
router.delete('/clientes/:id', service.deleteDataCliente);
```

**Controller**: `reservaionFronEnd/controllers/clientes.controllers.js`

The controller executes direct SQL queries against the `cliente` table:
- `SELECT * FROM cliente` - Get all clients
- `INSERT INTO cliente (nombre, correo, telefono) VALUES (?, ?, ?)` - Create client
- `UPDATE cliente SET nombre=?, correo=?, telefono=? WHERE id=?` - Update client
- `DELETE FROM cliente WHERE id=?` - Delete client

</Expandable>

## Database Schema

The `cliente` table structure:

| Column   | Type         | Description                    |
|----------|--------------|--------------------------------|
| id       | INTEGER      | Primary key, auto-increment    |
| nombre   | VARCHAR      | Client's full name             |
| correo   | VARCHAR      | Client's email address         |
| telefono | VARCHAR      | Client's phone number          |
